Array

En esta seccion encontrarás informacion sobre que es un array

Un array es una estructura de datos que permite almacenar múltiples elementos del mismo tipo (por ejemplo, enteros, cadenas de texto, etc.). Todos los elementos están almacenados en posiciones específicas llamadas índices. Los índices comienzan en 0, es decir, el primer elemento tiene el índice 0, el segundo el 1, y así sucesivamente. El tamaño del array es fijo, es decir, una vez definido no puede cambiar.

Declaración y creación de arrays:

int[] codigoColores = new int[4]; // [0, 0, 0, 0]

Aquí estamos declarando un array llamado código de Colores que puede almacenar 4 enteros. Todos los elementos del array se inicializan en 0 de forma predeterminada porque es un array de enteros.

Array List

En esta seccion encontrarás informacion sobre que es un array list

Un ArrayList en Java es una clase que pertenece al paquete java. Es una lista ordenada y dinámica que: Permite elementos duplicados. Respeta el orden de inserción (es decir, se mantienen los elementos en el orden en que fueron agregados). Automáticamente ajusta su tamaño conforme se agregan o eliminan elementos. Ofrece métodos útiles como add(), remove(), size(), get(), etc.

Sintaxis general:

ArrayList nombreVariable = new ArrayList<>();

Java no permite usar tipos primitivos (int, double, etc.) directamente en un ArrayList, por eso usamos las clases wrapper como Integer, Double, etc.

Declaración de ArrayList:

ArrayList numeros; // Lista de números enteros ArrayList nombreEtiquetas; // Lista de cadenas de texto

HashMap

En esta seccion encontrarás informacion sobre que es un HashMap.

Un HashMap es una estructura de datos que almacena pares clave-valor. Es útil cuando necesitas encontrar rápidamente un valor usando una clave única. Algunos puntos clave son: Las claves (keys) deben ser únicas (no se pueden repetir). Los valores (values) pueden repetirse. No garantiza ningún orden específico de los elementos. Ofrece una forma eficiente de almacenar y buscar datos.

Instanciar un HashMap:

HashMap funkos = new HashMap<>();

Aquí estamos creando un HashMap donde las claves son enteros (Integer) y los valores son cadenas de texto (String).

Agregar elementos con put()

funkos.put(1, "Batman"); funkos.put(1329, "Spiderman");

put(key, value) agrega un par clave-valor al mapa. Ejemplo: Clave: 1 → Valor: "Batman" Clave: 1329 → Valor: "Spiderman" Si agregas una clave que ya existe, el valor anterior se reemplaza con el nuevo.

Ver el contenido del mapa

System.out.println(funkos);

{1=Batman, 1329=Spiderman, 875=Prision Mike, ...}

Resumen HashMap para almacenar pares clave-valor y realizar las operaciones más comunes como agregar, eliminar, reemplazar, y buscar elementos. También te enseña a iterar sobre las claves y los valores, y cómo ordenar las claves con TreeMap. Estos conceptos son esenciales para trabajar con estructuras de datos en Java, y dominar estos métodos te permitirá manejar colecciones de manera eficiente en tus programas.

Clase

En esta seccion encontrarás informacion sobre las clases:

Una clase en Java es una plantilla o blueprint para crear objetos. Dentro de una clase se definen: Atributos: Son variables que describen las propiedades de los objetos. Métodos: Son funciones que definen el comportamiento de los objetos.

Atributos (Variables)

Los atributos pueden ser: No estáticos (non-static): Cada objeto creado a partir de la clase tendrá su propia copia de estas variables. Estáticos (static): Se comparten entre todos los objetos de la clase, y pertenecen a la clase en sí, no a una instancia específica.

String name; // Atributo no estático (propiedad individual de cada objeto) int age; // Atributo no estático int food; // Atributo no estático static String specie; // Atributo estático (compartido por todos los gatos)

Métodos: Estáticos y No Estáticos

Método no estático (non-static) Un método no estático necesita que se cree un objeto de la clase para poder ser utilizado.

int portions(int amount) { return food += amount; }

Método estático (static) Un método estático pertenece a la clase en sí, no a ninguna instancia. Esto significa que se puede llamar sin necesidad de crear un objeto.

static String specie() { return "Bienvenido a Petco! \\Tenemos: " + specie; }

En esta seccion encontrarás informacion sobre los 4 pilares de la Programación Orientada a Objetos:

  • Encapsulación:
  • Herencia:
  • Polimorfismo:
  • Abstracción

Encapsulación: Oculta los detalles internos de un objeto y controla cómo otros objetos acceden a su información. Herencia: Permite que una clase reutilice atributos y métodos de otra clase (por ejemplo, un cajero ATM que herede de una clase general Maquina). Polimorfismo: Habilidad de que un objeto se comporte de múltiples maneras (por ejemplo, un método que funciona de forma diferente en subclases). Abstracción: Define qué hace un objeto sin especificar cómo lo hace en detalle. Los cuatro pilares de OOP son fundamentales para estructurar un programa eficiente y mantenible.
